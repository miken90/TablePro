# Thiết lập Phát triển

## Yêu cầu

### Phần mềm bắt buộc

| Phần mềm | Phiên bản | Mục đích |
|----------|---------|---------|
| **macOS** | 14.0+ (Sonoma) | Nền tảng target |
| **Xcode** | 15.0+ | IDE phát triển |
| **Git** | Latest | Quản lý phiên bản |

### Công cụ tùy chọn

| Công cụ | Mục đích |
|------|---------|
| **SwiftLint** | Code linting |
| **SwiftFormat** | Code formatting |
| **Homebrew** | Quản lý package |
| **Git LFS** | Pull thư viện native từ `Libs/` |

## Bắt đầu

### Bước 1: Clone và Bootstrap

```bash
git clone https://github.com/datlechin/tablepro.git
cd tablepro

# Pull thư viện native (lưu trong Git LFS)
brew install git-lfs
git lfs install
git lfs pull

# Tạo file config bắt buộc (file trống đủ cho phát triển)
touch Secrets.xcconfig

# Cài đặt công cụ lint/format
brew install swiftlint swiftformat
```

<Warning>
Bỏ qua `git lfs pull` sẽ khiến `Libs/` chứa file con trỏ thay vì binary, gây lỗi linker.
</Warning>

### Bước 2: Mở trong Xcode

```bash
open TablePro.xcodeproj
```

### Bước 3: Build và Run

1. Chọn scheme **TablePro**
2. Chọn **My Mac** làm destination
3. Nhấn `Cmd+R` để build và chạy

## Build

### Build Phát Triển

```bash
# Build kiến trúc hiện tại (Debug)
xcodebuild -project TablePro.xcodeproj -scheme TablePro -configuration Debug build -skipPackagePluginValidation

# Hoặc trong Xcode: Cmd+B
```

### Build Release

```bash
# Apple Silicon
scripts/build-release.sh arm64

# Intel
scripts/build-release.sh x86_64

# Cả hai kiến trúc
scripts/build-release.sh both
```

Xem [Build](/vi/development/building) để biết chi tiết.

## Chạy Tests

```bash
# Tất cả tests
xcodebuild -project TablePro.xcodeproj -scheme TablePro test -skipPackagePluginValidation

# Test class cụ thể
xcodebuild -project TablePro.xcodeproj -scheme TablePro test -skipPackagePluginValidation \
    -only-testing:TableProTests/TestClassName

# Test method cụ thể
xcodebuild -project TablePro.xcodeproj -scheme TablePro test -skipPackagePluginValidation \
    -only-testing:TableProTests/TestClassName/testMethodName
```

Hoặc trong Xcode: **Product** > **Test** (`Cmd+U`).

## Chất lượng Code

### SwiftLint

```bash
# Kiểm tra vấn đề
swiftlint lint

# Tự động sửa
swiftlint --fix
```

SwiftLint cũng chạy tự động khi build Xcode.

### SwiftFormat

```bash
# Format tất cả code
swiftformat .

# Kiểm tra mà không áp dụng
swiftformat --lint .
```

<Tip>
Chạy SwiftFormat trước khi commit để giữ code style nhất quán.
</Tip>

## Tệp cấu hình

### .swiftlint.yml

Rules và cấu hình SwiftLint:

- Giới hạn độ dài dòng
- Giới hạn độ phức tạp function
- Rules bị vô hiệu hóa
- Rules tùy chỉnh

### .swiftformat

Tùy chọn SwiftFormat:

- Kiểu indentation
- Vị trí brace
- Thứ tự import
- Các rules formatting khác

## Workflow phát triển

### Thực hiện thay đổi

1. Tạo feature branch:
   ```bash
   git checkout -b feature/my-feature
   ```

2. Thực hiện thay đổi

3. Chạy linting:
   ```bash
   swiftlint lint
   swiftformat --lint .
   ```

4. Chạy tests:
   ```bash
   xcodebuild -project TablePro.xcodeproj -scheme TablePro test -skipPackagePluginValidation
   ```

5. Commit:
   ```bash
   git add .
   git commit -m "Add my feature"
   ```

6. Push và tạo pull request:
   ```bash
   git push origin feature/my-feature
   ```

### Debugging

1. Đặt breakpoints bằng cách click số dòng
2. Dùng `po` trong debug console để in objects
3. Dùng Variables View để kiểm tra state
4. Dùng Console.app cho system logs

### Logging

TablePro dùng OSLog:

```swift
import os

private static let logger = Logger(subsystem: "com.TablePro", category: "ComponentName")

// Sử dụng
logger.debug("Connection established")
logger.info("Query executed in \(duration)s")
logger.error("Failed to connect: \(error.localizedDescription)")
```

## Tác vụ thông dụng

### Thêm View mới

1. Tạo SwiftUI view trong subfolder `Views/` phù hợp
2. Theo quy ước đặt tên (`FeatureNameView.swift`)
3. Dùng `@Observable` hoặc `@StateObject` cho state management
4. Thêm vào navigation/routing nếu cần

### Thêm tính năng Database

1. Tạo plugin bundle mới trong `Plugins/`, hoặc sửa driver plugin hiện có
2. Triển khai các method trong protocol `PluginDatabaseDriver`
3. Cập nhật `PluginDriverAdapter` nếu cần bridge method mới
4. Thêm UI trong view components liên quan
5. Thêm tests

### Sửa đổi Settings

1. Thêm vào struct settings trong `AppSettings.swift`
2. Cập nhật `SettingsView`
3. Dùng setting trong app qua `@AppStorage` hoặc settings manager

## Khắc Phục Sự Cố

### Lỗi Build

1. Clean build folder: `Cmd+Shift+K`
2. Xóa derived data:
   ```bash
   rm -rf ~/Library/Developer/Xcode/DerivedData
   ```
3. Đóng và mở lại Xcode
4. Kiểm tra cập nhật Xcode

### Lỗi SwiftLint

1. Đọc error message
2. Kiểm tra `.swiftlint.yml`
3. Sửa vấn đề hoặc disable rule nếu phù hợp
4. Chạy `swiftlint --fix` cho vấn đề auto-fix được

### Thiếu thư viện hoặc Secrets.xcconfig

Chạy `git lfs pull` và `touch Secrets.xcconfig` tại thư mục gốc. Xem [Bước 1](#bước-1-clone-và-bootstrap).
